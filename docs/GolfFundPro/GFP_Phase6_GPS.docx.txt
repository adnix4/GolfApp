1. Phase 6 Goal & Scope
Phase 6 adds GPS-based distance to pin, elevation change display, and Mapbox satellite hole view to the mobile scoring app. It also enables GPS-based drive distance tracking for longest drive contests. Phase 6 is kept separate because it introduces genuinely different technical requirements: PostGIS spatial queries, Mapbox GL integration, DEM terrain elevation APIs, and on-course device GPS behavior.

2. Cup Location Setup
2.1  Admin Course Setup Tool
* New screen in mobile app: Course Setup (admin role only). Lists all 18 holes.
* Admin taps a hole → 'Set Pin Location' button. App takes current GPS reading.
* Device GPS averages 5 readings over 5 seconds to reduce error. Shows uncertainty radius (± N meters).
* Admin confirms. PATCH /api/v1/courses/{id}/holes/{holeNumber}/cup → saves to course_holes.cup_location.
* DEM elevation fetched server-side after save: GET https://api.opentopodata.org/v1/srtm30m?locations={lat},{lng}. Saved to cup_elevation_m.
* One-time setup per course. Admin can update pin position day-of (pin moved for the round).
* PostGIS index: CREATE INDEX ON course_holes USING GIST (cup_location) — already in Phase 1 migration, now matters.


3. Distance to Pin
3.1  Offline-First Calculation
* Cup coordinates downloaded to SQLite at event/round join as part of event_cache. < 5 KB for all 18 holes.
* Online mode: GET /api/v1/holes/{id}/distance?lat={lat}&lng={lng} → server uses ST_Distance(cup_location, ST_SetSRID(ST_MakePoint(lng,lat),4326)) in meters, converted to yards.
* Offline mode (primary): client-side Haversine formula. < 1ms. No network required.

// Haversine in TypeScript (packages/ui/utils/geo.ts)
export function distanceYards(lat1: number, lng1: number,
                             lat2: number, lng2: number): number {
 const R = 6371000; // Earth radius in meters
 const dLat = toRad(lat2 - lat1);
 const dLng = toRad(lng2 - lng1);
 const a = Math.sin(dLat/2)**2
         + Math.cos(toRad(lat1)) * Math.cos(toRad(lat2))
         * Math.sin(dLng/2)**2;
 const meters = R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
 return meters * 1.09361; // meters to yards}


3.2  Elevation Change
* Player elevation: fetched from DEM API at player's current GPS coordinates. Cached for 60 seconds (player doesn't move that fast).
* Elevation change = cup_elevation_m − player_elevation_m. Positive = uphill, negative = downhill.
* Display: '+12 ft uphill' or '−8 ft downhill' below the yardage number.
Never use device GPS altitude:  Device barometric/GPS altitude has ±10–30 ft accuracy — useless for golf elevation. Always use DEM terrain data fetched from a tile server or API.


3.3  GPS Polling
* expo-location watchPositionAsync: accuracy = HIGH, timeInterval = 15000 (15 seconds), distanceInterval = 5 (5 meters).
* Polling paused automatically when score entry modal is open (saves battery, player isn't moving).
* Polling paused when app goes to background.
* Uncertainty indicator: if GPS accuracy > 10 meters, show ± ring around player dot on map.

4.2  Offline Tile Cache
* Before round starts: mandatory pre-flight item — 'Course map downloaded'
* MapboxGL.offlineManager.createPack({ name: courseId, styleURL: Satellite, bounds: courseBounds, minZoom: 14, maxZoom: 18 })
* courseBounds: bounding box of all 18 holes' cup locations + 500m padding. Calculated from cup_location data in event_cache.
* Estimated download size: ~150–250 MB for an 18-hole course at zoom 14–18. Download on WiFi before leaving for course.
* Pack stored in device filesystem. Persists across app restarts. Re-download only if cup locations change.


5. GPS Drive Distance Tracking
* 'Mark Drive' button appears on hole screen after tee shot is expected (hole 1 tee, etc.).
* Scorekeeper taps 'Mark Drive' when team selects a drive. GPS coordinates recorded and stored in pending_scores as drive_location.
* Scorekeeper taps 'Mark Ball' on arrival at ball. GPS coordinates stored as ball_location.
* Distance calculated: distanceYards(driveLat, driveLng, ballLat, ballLng). Displayed with ± disclaimer.
* Manual entry always available: if GPS is unreliable (wooded hole), scorekeeper enters drive distance in yards via text input.
* Per-hole config: wooded_hole flag in course setup disables GPS drive distance for that hole and defaults to manual.
* Challenge leaderboard: longest drive_location-to-ball_location distance per hole surfaced in hole challenge tab.


6. Phase 6 API Endpoints
Method
	Endpoint
	Description
	PATCH
	/courses/{id}/holes/{n}/cup
	Save cup GPS coordinates and trigger DEM elevation fetch.
	GET
	/holes/{id}/distance
	Server-side distance calculation. ?lat=&lng=. Returns yards + elevation change.
	GET
	/events/{id}/course-cache
	Download full course GPS data for SQLite: all cup locations, hole metadata, course bounds for tile download.
	PATCH
	/scores/{teamId}/{holeNumber}/drive
	Save drive_location and ball_location from GPS. Returns calculated drive distance.
