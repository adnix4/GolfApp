1. Phase 1 Goal & Scope
Phase 1 delivers a complete tournament management system the organizer can use at a real event. Golfers register online and check in via QR code. The organizer enters scores manually from a tablet. There is no mobile scoring app yet — that is Phase 2. An admin-only leaderboard shows standings. A branded public landing page accepts registrations and donations.


Deliverable:  An organizer can run the CLHS 4th Annual Golf & Give end-to-end using Phase 1 alone. Every subsequent phase adds capability without replacing any Phase 1 work.

3. Organization & Event Setup
3.1  Organization Management
* POST /api/v1/auth/register — creates user + organization in one call. Returns JWT pair.
* GET/PATCH /api/v1/orgs/{id} — org name, slug, logo upload, mission statement, is_501c3, theme tokens
* Logo upload: POST /api/v1/orgs/{id}/logo — multipart/form-data. API resizes to 200px height, stores on CDN, returns URL.
* Theme save: validateContrast(primary, surface) from packages/theme runs server-side before saving. Returns 422 if contrast < 4.5:1.


3.2  Event Creation
* POST /api/v1/events — creates event under org. Returns event_code (8-char random alphanumeric).
* Format: scramble | stroke | stableford | best_ball
* Start type: shotgun (all teams start simultaneously on different holes) | tee_times (sequential)
* Holes: 9 or 18. All downstream calculations adapt.
* Shotgun setup: POST /api/v1/events/{id}/shotgun-assignments — auto-assigns teams to holes, validates no duplicates.
* Tee time setup: POST /api/v1/events/{id}/tee-times — start time + interval in minutes. Returns full schedule.
* Course setup: POST /api/v1/events/{id}/course — attach course record, set par and handicap_index per hole.


3.3  Hole Challenge Configuration
* POST /api/v1/events/{id}/challenges — create challenge: type, hole_number (null = all-day challenge), description, prize, sponsor_id
* PATCH /api/v1/challenges/{id}/result — admin enters result manually (distance in yards, player name). Phase 6 upgrades to GPS measurement.


4. Three-Mode Registration System
All three registration modes are built and enabled in Phase 1. Each mode can be independently toggled per event via events.config.


4.1  Mode 1 — Full Team Registration
* POST /api/v1/events/{id}/register/team — captain provides team name, all player details, and entry fee method
* Invite links (optional): POST /api/v1/teams/{id}/invite — generates a signed invite_token (HMAC, 48hr expiry). Sends email with registration link.
* Teammate self-registers via /e/{orgSlug}/{eventCode}/join?token={invite_token}
* Slot tracking: teams.max_players (default 4). When all slots filled, team.is_complete flips to true automatically.


4.2  Mode 2 — Join an Existing Team
* GET /api/v1/events/{id}/teams?q={search} — fuzzy name search returns open teams with available slots
* POST /api/v1/events/{id}/register/join — { team_id, player details }. Validates slot availability with optimistic lock.
* Captain receives email notification when a new player joins their team.


4.3  Mode 3 — Free Agent Pool
* POST /api/v1/events/{id}/register/free-agent — player details + preferences: skill_level (beginner|intermediate|advanced|competitive), age_group (under30|30to50|over50), gender, pairing_note
* GET /api/v1/events/{id}/free-agents — admin view: all unassigned players with preference cards
* POST /api/v1/events/{id}/free-agents/assign — { player_id, team_id }. Admin drag-and-drop. Triggers team assignment email.
* POST /api/v1/events/{id}/free-agents/auto-pair — handicap snake draft algorithm. Returns proposed pairings for admin review before committing.
* Auto-pair algorithm: sort free agents by handicap, snake-draft into groups of max_players. Soft preference matching (same age group) applied as tiebreaker.


5. Admin Dashboard (Expo Router Web)
The admin dashboard is a web app built with Expo Router. It shares ScoreCard, TeamCard, and other components from packages/ui with the Phase 2 mobile app. Organizers access it from any browser on a tablet, laptop, or phone.


5.1  Dashboard Screens — Phase 1
* Event Hub (/admin/events/{id}) — Central screen: quick stats (teams checked in, scores complete, $ raised), nav to all panels below
* Registration Panel — Tabs: Complete Teams | Partial Teams | Free Agents | Pending Invites. Each team card shows players, payment status, and check-in status.
* Free Agent Board — Kanban-style: Unassigned pool on left, team cards on right. Drag-and-drop assignment. Auto-pair button with preview modal.
* Check-In Screen — Full-screen QR scanner (device camera) or name search. Individual and team check-in. Live counter widget.
* Score Entry — Select team → enter hole-by-hole gross scores. Running to-par displayed. Save auto-triggers leaderboard recalculation.
* Leaderboard (Admin) — Sortable standings table: rank, team, to-par, gross, thru. Not publicly accessible in Phase 1.
* Hole Challenges — Manual result entry per challenge. Closest-to-pin: enter distance. Longest drive: enter distance. Puts result on challenge leaderboard.
* Sponsors Panel — Upload logos, set tier and placements. Preview how sponsor appears on landing page and (in Phase 2) leaderboard.
* Fundraising Panel — Entry fees collected vs. expected. Donation log. Grand total counter.
* Print Kit — Button: Generate QR Kit PDF. Button: Generate Paper Scorecards PDF (one per team, pre-filled with hole, par, handicap_index).


6. QR Check-In
* Every registered player receives a unique QR code in their confirmation email. The QR encodes a signed token: { type: player_checkin, player_id, event_id, signature }
* Admin check-in screen: camera streams to QR decoder. On successful scan, PATCH /api/v1/players/{id}/checkin called. Player card flips to checked-in state with timestamp.
* Team check-in shortcut: if all players in a team are checked in, team.check_in_status flips to complete automatically.
* Walk-up registration: from check-in screen, admin taps 'Add Walk-Up'. Abbreviated form (name, email, team assignment). Generates and displays QR on screen for player to screenshot.
* Live counter component subscribes to check-in events (polling in Phase 1; SignalR in Phase 3). Shows 'X of N teams checked in'.

Email Timing:  Confirmation sends immediately on registration. Reminder sends via a scheduled job at event.start_at - 24 hours. Thank-you sends when admin marks event status = completed. Use Hangfire (background jobs library for .NET) for scheduled sends.


8. Public Landing Page (Next.js)
* Route: /e/{orgSlug}/{eventCode} — SSR rendered. Available in Phase 1 as a registration page. Leaderboard link appears in Phase 2.
* Above fold: org logo, event name, date, course name. Org theme colors applied via CSS custom properties.
* Registration CTAs: three buttons — Register a Team, Join a Team, I Need a Team. Each links to the relevant registration form.
* Donation widget: DonationForm component with amount presets and custom amount. POST /api/v1/events/{id}/donations. Thermometer bar shows progress toward goal.
* Sponsor showcase: logos displayed in tier order (Title → Gold → Hole → Silver → Bronze).
* Remaining slots counter: 'X team spots remaining' updated server-side on each page load (ISR, 60-second revalidate).
* JSON-LD Event structured data in <head> for Google Search rich results.
* Open Graph meta tags: event name, date, org logo — for social media sharing.
* 'Leaderboard Coming Soon' placeholder shown in Phase 1. Replaced with live leaderboard link in Phase 2.
